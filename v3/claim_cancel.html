Cancel claim

POST remote/api/v3/claim/{claim_id}/cancel

id - claim id from claims list page or view purchase or view lot

purchase - state_purchase uid from view purchase

purchase_lot - state_purchase_lot uid from view lot

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible request fields:

  • reason - type: str, required
  • purchase - type: str, required
  • purchase_lot - type: str, optional if purchase passed

possible response data fields:

  • id - type: str

Examples

request:

clear && curl --header "Acc-Token: auth_token" --header "Api-Agent: zk_ios" --header "Content-Type: application/json" -X POST "http://my.zakupki.example.com:8000/remote/api/v3/claim/485ae874ce5511e98979cf44cd9c0a51/cancel" --data @request_data

response:

{
  "status": "ok",
  "errors": [],
  "data": {
    "id": "485ae874ce5511e98979cf44cd9c0a51"
  },
  "system": ""
}
