Store users device info data

POST /remote/api/v3/device_info

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible request data params:

  • device_model - type: string, required
  • os - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

successful request always returns empty "ok" response

Examples:

request:

clear && curl -X POST "http://my.example.com:5001/remote/api/v3/device_info" --header "Content-Type: application/json" --header "Api-Agent: zk_ios" --header "Acc-Token: 2806394842ede6b643d4fd99eb9486c41753604d" --data '{"a": "b"}' | json_pp

response:

{
   "errors" : [
      {
         "field" : "device_model",
         "messages" : "Це поле є обов`язковим"
      },
      {
         "field" : "os",
         "messages" : "Це поле є обов`язковим"
      }
   ],
   "status" : "error",
   "system" : ""
}

request:

clear && curl -X POST "http://my.example.com:5001/remote/api/v3/device_info" --header "Content-Type: application/json" --header "Api-Agent: zk_ios" --header "Acc-Token: 2806394842ede6b643d4fd99eb9486c41753604d" --data '{"device_model": "dssss", "os": "osss"}' | json_pp

response:

{
   "data" : {},
   "errors" : [],
   "status" : "ok",
   "system" : ""
}
