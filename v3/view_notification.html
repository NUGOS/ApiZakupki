View single notification

GET remote/api/v3/notifications/{notification_id}

notification_id - _id of notification, can be find in notifications list

request headers:

  • Api-Agent - type: string, required
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible response data fields: see notification object fields from notifications list

Example

request:

clear && curl -X GET "http://my.example.com:5001/remote/api/v3/notifications/5d517fdb1f18015f2813522b" --header "Acc-Token: 43b06b921e3df14a66cb9a38be51e71dabde5856" --header "Api-Agent: zk_ios" | json_pp
