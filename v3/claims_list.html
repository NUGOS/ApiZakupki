Claims list for tender

GET /remote/api/v3/claims/{uid}

uid - purchase uid from view, search

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

response data fields

  • complaints - type: array of complaints objects or empty, see claim view
  • lots - type: array of lot complaints objects or empty, described below
  • permissions: - type: complaints permissions object, described below
  • uid - type: str
  • url - type: str

lot complaints object fields

  • complaints - type: array of complaints objects or empty, see claim view
  • uid - type: str
  • url - type: str

permission object data fields

  • award_choices - type: array of award choice objects or empty, described below
  • can_create - type: bool
  • create_scope - type: int
  • create_title - type: str
  • create_until - type: date str or null, format yyyy-mm-ddTHH:MM:SS.fZ
  • id - type: str
  • lots - type: array of lot permission objects or empty, described below
  • qualification_choices - type: array of qualification choice objects or empty, described below

lot permission object data fields

  • award_choices - type: array of award choice objects or empty, described below
  • can_create - type: bool
  • create_until - type: date str or null, format yyyy-mm-ddTHH:MM:SS.fZ
  • id - type: str
  • qualification_choices - type: array of qualification choice objects or empty, described below

award choice object fields

  • id - type: str (award id)
  • name - type: str (merchant name)

qualification choice object fields

  • id - type: str (qualification id)
  • name - type: str (merchant name)

award choices and qualification choices are needed to create a claim or complaint on specific supplier during "prequalification_stand_still" and "qualification" tender statuses (see create_claim doc)

request:

clear && curl -X GET --header "Acc-Token: 5a119647c3fa191ab60d41ed9fabf784453999d4" --header "Api-Agent: zk_ios" "http://my.example.com:5001/remote/api/v3/claims/ae54ad50899147e88fa5675917288e78" | json_pp
