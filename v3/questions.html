List of tender questions

GET /remote/api/v3/questions/{uid}

uid - purchase uid from purchase view or purchase search

request headers:

  • Api-Agent - type: string, required
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

response data fields:

  • id - type: str
  • lots - type: array of lot questions objects or empty, described below
  • permissions - type: permission object, see below
  • questions - type: array of question objects or empty, described below

lots questions object fields:

  • id - type: str
  • questions - type: array of question objects or empty, described below

question object fields:

  • answer - type: str or null
  • author - type: merchant object or empty, see tender view
  • date - type: date string, format YYYY-MM-DDTHH:MM:SS or null
  • date_answered - type: date string, format YYYY-MM-DDTHH:MM:SS or null
  • description - type: str
  • question_of - type: str
  • title - type: str
  • uid - type: str

permissions object fields:

  • can_create - type: bool
  • create_until - type: date string, format YYYY-MM-DDTHH:MM:SS or null
  • id - type: str
  • lots - type: array or permission objects for lots or empty

request:

clear && curl -X GET "http://my.example.com:5001/remote/api/v3/questions/61e9ac0e68668b6b1701c7173e6b2048a831727e" --header "Acc-Token: e53e0e4a34784eb359a9abba73b81927e2cb48fa" --header "Api-Agent: zk_ios" | json_pp
