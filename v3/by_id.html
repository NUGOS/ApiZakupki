Get API endpoint by tender id or tender id + lot-id

GET remote/api/v3/by_id/{id}{tail}

id - purchase id (postgres primary key) from view or search or string: 'lot'

tail - can contain any valid url string (example: "/208066" or "/lot-208066")

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible data fields:

  • tender - type: str
  • lot - type: str or null

Examples:

request:

clear && curl -X GET --header "Acc-Token: 5a119647c3fa191ab60d41ed9fabf784453999d4" --header "Api-Agent: zk_ios" "http://my.example.com:5001/remote/api/v3/by_id/2707544" | json_pp

response:

{
   "data" : {
      "lot" : null,
      "tender" : "7f62165f687f4c4bafd3f2d8f38c7f7a"
   },
   "errors" : [],
   "status" : "ok",
   "system" : ""
}
