Get api data for claim/complaint by portal/cabinet id/uid

GET remote/api/v3/dispatch_complaint/{cid}

cid - claim/complaint uid/id from cabinet/portal url

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible data fields:

  • lot_uid - type: str or null
  • tender_uid - type: str or null
  • type - type: str, possible values one of ["claim", "complaint"]
  • uid - type: str

Examples:

request:

clear && curl -X GET "http://my.example.com:5001/remote/api/v3/dispatch_complaint/848d6d54d5ba451bb85226580a854a25" --header "Acc-Token: 3010d1bfb81de262d23aa793c5c9fd972e4b93ba" --header "Api-Agent: zk_ios" | json_pp

response:

{
   "data" : {
      "lot_uid" : null,
      "tender_uid" : "72154f78d55545c7a42082f714d6d9a2",
      "type" : "claim",
      "uid" : "848d6d54d5ba451bb85226580a854a25"
   },
   "errors" : [],
   "status" : "ok",

}
