Api-Agent header

Since we allow use api for some 3rd party integrations we need some tool for blocking api usage.

As a simplest one - we have add another "secret" header to api call.

Each api call should provide Api-Agent header with special value

api agent header value example
IOS zk_ios Api-Agent: zk_ios
Android zk_android Api-Agent: zk_android
Custom integrations zk_integration Api-Agent: zk_integration
Bitrix24 zk-bitrix24 Api-Agent: zk-bitrix24
Zk_zimmer zk_zimmer Api-Agent: zk_zimmer

If client does't provide Api-Agent header api will response with 406 http response.

During some preriod we will allow api calls without this header
