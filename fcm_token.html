Push уведомления

реистрация клиентского токена (firebase cloud messaging registration token)

для получения приложением push уведомлений

POST /remote/api/v1/fcm

request headers:

  • Api-Agent - type: string, reuqired
  • Acc-Token - type: string, required

possible request arguments:

  • fcm_token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

Уведомления генерируемые сервером

На данный момент сервер генерирует 3 типа пуш уведомлений:

  • info_question_answered - ответ на вопрос в закупке
  • info_note_publication - системное уведомление
  • state_purchase_edited_event - произошли изменения в закупке, в которую пользователь подавал предложение
  • auction_start_soon_lot_push_event - уведомление о скором начале аукциона в лоте
  • auction_start_soon_push_event - уведомление о скором начале аукциона в тендере

Структура/содержание уведомлений

{
    "from": "XXXX",
    "collapse_key": "XXXXX", 
    "data": {
        "action": "http://zakupki.example.com:6544/gov/tenders/UA-2016-09-14-000019/questions",
        "action_data": {}
    }, 
    "priority": "high", 
    "notification": {
        "body": "Ви отримали відповідь на своє запитання UA-2016-09-14-000019"
    }
}
  • data - используется сервером для передчи информации о необходимом действии при нажатии на уведомление само действие содержится в поле action, дополнительная информация содержится в поле action_data
  • action_data - опциональное поле, может содержать объект или быть null
  • notification - текст самого уведомления передается всегда в поле body

возможное содержимое поля action

Планируется два возможных варианта действий:

  • открытие ссылки в браузере
  • открытие приложения по заданному пути

Разница в действиях определеятся протоколом в поле action. Так, для открытия ссылки в браузере action будет начинаться с http(s)://, для действий в приложении - app://

Пример с app протоколом:

{
    "from": "XXXX",
    "collapse_key": "XXXXX", 
    "data": {
        "action": "app://tender/1233/questions"
    }, 
    "priority": "high", 
    "notification": {
        "body": "Ви отримали відповідь на своє запитання UA-2016-09-14-000019"
    }
}

данный action описывает открытие раздела вопросы для тендера с указамым в пути id

v3 modifications

action_data field can cantain additional data for new v3 api resolvers

content of action_data field for each push notification described below

info_question_answered

  • event_name - str
  • purchase_uid - str
  • lot_uid - str or null
  • uid - str

auction_start_soon_lot_push_event

  • event_name - str
  • purchase_uid - str
  • lot_uid - str

auction_start_soon_push_event

  • event_name - str
  • purchase_uid - str

  • event_name - str
