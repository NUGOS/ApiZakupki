request headers:

  • Acc-Token - type: string, required

possible request arguments:

  • page - type: int
  • timestamp - type: int or empty

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible response data fields:

  • next_page - type: int or null
  • prev_page - type: int or null
  • notifications - type: array of notification objects or empty, described below
  • timestamp - type: int
  • current_page - type: int
  • total_pages - type: int
  • unread_count - type: int

notification object fields:

  • body_en - type: str (could contain HTML code)
  • body_ru - type: str (could contain HTML code)
  • body_uk - type: str (could contain HTML code)
  • status - type: string
  • event_id - type: str
  • is_read - type: boolean
  • created_datetime - type: datetime str, format YYYY-MM-DD HH:MM:SS.f
  • disable - type: boolean - if value set to false notification could be marked as read and back
  • service_id - type: int
  • delivery_status - type: str
  • _id - type: str
  • type - type: sty
  • sent_datetime - type: datetime str, format YYYY-MM-DD HH:MM:SS.f
  • event_name - type: str

curl -X GET --header "Acc-Token: eyJlbXBsb3llZSI6NjEyNjEsIm1lcmNoYW50Ijo1NTEzOH0.C8di6Q.JPtyyp32oH7KaM7WqyjQPE0SoXA" -X GET "http://my.example.com:5001/remote/api/v1/notifications?\
timestamp=1493907357&\
page=2"

Кол-во непрочитанных уведомлений

GET remote/api/v1/unread_notifications_count

request headers:

  • Acc-Token - type: string, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible response data fields:

  • count - type: int

request example:

curl -X GET --header "Acc-Token: eyJlbXBsb3llZSI6NjEyNjEsIm1lcmNoYW50Ijo1NTEzOH0.C8di6Q.JPtyyp32oH7KaM7WqyjQPE0SoXA" -X GET "http://my.example.com:5001/remote/api/v1/unread_notifications_count"

response example:

{  
    "status":"ok",
    "errors":[],
    "data":{  
        "count":3
    },
    "system":[]
}

Сделать прочинанным/не прочитанным

POST remote/api/v1/notifications/toggle_read?notification_id={_id}

_id - id уведомления

request headers:

  • Acc-Token - type: string, required

request params:

  • notification_id - type: str, required

possible response fields:

  • status - type: str, values: ok/error
  • errors - type: array of objects, validation errors
  • data - type: object
  • system - type: str, system error messages

possible response data fields:

  • response data always empty

request example:

curl -X POST --header "Acc-Token: eyJlbXBsb3llZSI6NjEyNjEsIm1lcmNoYW50Ijo1NTEzOH0.C8di6Q.JPtyyp32oH7KaM7WqyjQPE0SoXA" -X GET "http://my.example.com:5001/remote/api/v1/notifications/toggle_read?notification_id=590ad7847d90be953bb1623b"

response example:

{  
    "status":"ok",
    "errors":[],
    "data":{},
    "system":[]
}
